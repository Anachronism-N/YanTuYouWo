### 推荐排序

**第一推荐：研舟 YanZhou**

- Slogan：**「以研为舟，渡你上岸」**
- 理由：「研」字直接关联研究生，零认知成本（不需要解释"哪个 yan"）；「舟」保留了渡船意象，与 IP 形象「舟舟」天然呼应；YanZhou 发音清晰，域名 `yanzhou.com` / `yanzhou.app` 可探索；整体气质兼具学术感与温暖感。

**第二推荐：岸升 AnSheng**

- Slogan：**「一路向上，终达彼岸」**
- 理由：「上岸」是保研圈最高频的词汇，「岸升」把目标感拉满；口头传播极佳（"你用岸升了吗？"）；缺点是 IP 形象不太好关联。

**第三推荐：研策 YanCe**

- Slogan：**「研途万里，智策先行」**
- 理由：强调 AI 智能策略，适合后续产品定位向 AI 工具平台倾斜时使用；「策」字有策略、出谋划策之意，理性但不失温度。

## 第二部分：新功能详细设计

### 功能一：保研时间线（Baoyan Timeline）

#### 1.1 产品定位

保研时间线是用户主页的核心视图，根据用户画像自动生成一条个性化的保研全流程路线图。它将平台所有功能串联成「看得见的旅程」，是从"信息门户"升级为"保研操作系统"的关键抓手。

#### 1.2 用户故事

- 作为一名大三学生，我希望看到从现在到拿到 offer 的每个关键节点，这样我不会错过任何机会
- 作为一名刚注册的用户，我希望平台能根据我的情况自动规划路线，而不是自己到处翻信息
- 作为一名正在准备夏令营的学生，我希望时间线能提醒我即将到来的 deadline 和待办事项

#### 1.3 数据输入

时间线的个性化依赖以下用户数据（在 onboarding 或个人设置中收集）：

| 字段 | 来源 | 必填 |
|------|------|------|
| 当前年级 | onboarding | 是 |
| 本科院校 + 专业 | onboarding | 是 |
| 当前 GPA / 排名 | onboarding | 是 |
| 目标院校列表（1-5 所） | onboarding | 是 |
| 目标专业方向 | onboarding | 是 |
| 已有科研经历数 | onboarding | 否 |
| 已有竞赛/论文成果 | onboarding | 否 |
| 英语水平（CET-4/6 分数或雅思托福） | onboarding | 否 |

#### 1.4 时间线节点体系

根据保研标准流程，时间线包含以下阶段和节点类型：

**阶段一：背景提升期（大一秋 ~ 大三春）**

| 节点 | 时间 | 关联功能 | 动作 |
|------|------|----------|------|
| 确定保研意向 | 大一/大二 | → 新手引导 | 完成 onboarding |
| 参加竞赛 | 滚动（根据竞赛日历） | → /info/competitions | 查看推荐竞赛、收藏、日历提醒 |
| 联系导师进组科研 | 大二下/大三上 | → /info/tutors, /ai/tutor-match | 匹配导师、发送联系邮件 |
| 投递论文 | 大二下 ~ 大三下 | → /info/journals | 查看推荐期刊、投稿时间建议 |
| 英语考试 | 大二/大三 | → 知识库/备考技巧 | 查看备考资料 |
| GPA 阶段检查 | 每学期末 | → 背景画像雷达图 | 更新 GPA、查看竞争力变化 |

**阶段二：信息收集期（大三春 ~ 大三夏）**

| 节点 | 时间 | 关联功能 | 动作 |
|------|------|----------|------|
| 夏令营信息发布 | 3-5 月 | → /info/summer-camp | 浏览、筛选、收藏 |
| 准备申请材料 | 4-5 月 | → /ai/resume | 生成简历 |
| 文书撰写 | 4-5 月 | → 文书 AI 润色 | 撰写/润色个人陈述 |
| 择校定稿 | 5 月 | → /ai/recommend | AI 择校推荐 |

**阶段三：申请冲刺期（大三夏 ~ 大四秋）**

| 节点 | 时间 | 关联功能 | 动作 |
|------|------|----------|------|
| 夏令营申请提交 | 5-6 月 | → 任务管理 | 追踪各校申请状态 |
| 夏令营面试准备 | 6-7 月 | → /ai/interview | 模拟面试、问题预测 |
| 夏令营参加 | 7-8 月 | → 社区/经验分享 | 查看往届经验 |
| 预推免申请 | 8-9 月 | → /info/pre-admission | 浏览、申请 |
| 预推免面试 | 9 月 | → /ai/interview | 面试准备 |
| 推免系统填报 | 9-10 月 | → 任务管理 | 确认最终志愿 |
| 收到 offer | 10 月 | → 成就系统 | 解锁「上岸」成就 |

**视图模式：**

- **时间轴视图**（默认）：纵向时间轴，展示节点状态和关联操作

- deadline 前 7 天 / 3 天 / 1 天分级提醒

- 阶段完成时，舟舟发出庆祝动画

#### 1.6 技术方案

| 层级 | 方案 |
|------|------|
| 数据模型 | 新增 `TimelineNode` 类型：`{ id, phase, title, description, startDate, endDate, status, linkedRoute, linkedEntityIds, reminders[] }` |
| 生成逻辑 | 后端根据用户 profile + 目标院校的历史时间数据，调用规则引擎（优先）或 LLM 生成个性化时间线 |
| 存储 | 时间线数据持久化到用户 profile 关联表，支持手动编辑和 AI 重新生成 |
| 前端路由 | `/progress/timeline` 或复用 `/progress` 作为主入口 |
| 组件 | 新建 `TimelineView.tsx`、`TimelineNode.tsx`、`CalendarView.tsx` |
| 提醒 | 前端 `localStorage` 存储已读提醒状态；后续接入 Web Push Notification |

#### 1.7 MVP 范围

第一版：
- 基于 onboarding 数据生成静态时间线（规则引擎，非 LLM）
- 时间轴视图 + 节点状态手动更新
- 节点关联跳转到对应功能页

后续迭代：
- AI 动态调整时间线（根据进度和变化重新规划）
- 日历/看板视图
- 提醒推送

### 功能二：背景画像雷达图（Competitiveness Radar）

#### 2.1 产品定位

为用户生成一张直观的五维（或更多维度）竞争力雷达图，并与目标院校往年录取数据进行对比，让用户清楚地看到自己的优势和短板。这是一个**极强的获客工具**——"测测你的保研竞争力"天然具有社交传播属性。

#### 2.2 评估维度

| 维度 | 数据来源 | 计算逻辑 |
|------|----------|----------|
| **学业成绩** | GPA、排名百分比、核心课程成绩 | 归一化到 0-100：排名 Top 1% = 100, Top 5% = 90, Top 10% = 80, Top 20% = 65, Top 30% = 50, 其他递减 |
| **科研经历** | 科研项目数量、级别（国家级/省级/校级/实验室）、角色（负责人/参与者）、持续时间 | 加权评分：国家级项目负责人 = 30分/项，省级 = 20，校级 = 10，实验室 = 5；上限 100 |
| **论文成果** | 论文数量、级别（SCI 分区/CCF 等级/EI/核心/普刊）、作者排序 | 加权评分：SCI Q1 一作 = 40, SCI Q2 一作 = 30, CCF-A 一作 = 35, 二作减半；上限 100 |
| **竞赛获奖** | 竞赛数量、级别（国际/国家/省/校）、名次（金/银/铜/一二三等奖） | 加权评分：国际金 = 40, 国家一等 = 30, 国家二等 = 20, 省级 = 10；上限 100 |
| **综合素质** | 英语成绩、实习经历、学生工作/社会活动、推荐信 | CET-6 ≥ 550 = 25分，雅思 ≥ 7.0 = 30分；知名企业实习 = 20分/段；学生干部 = 10分；上限 100 |

#### 2.3 目标院校对比

从目标院校的历史录取数据（爬虫采集 + 社区用户回报 + 人工整理）中提取各维度的：
- **录取平均线**：往年录取学生的平均水平
- **录取最低线**：最低录取标准的估计
- **录取优势线**：往年录取学生中 Top 25% 的水平

在雷达图上同时展示用户数据和目标院校三条参考线，一目了然。

#### 2.5 社交传播设计

- **分享卡片**：生成一张品牌化的分享图（包含雷达图 + 综合评分 + 舟舟形象 + 平台二维码），可保存到相册或分享到微信/微博

### 功能三：信息差发现器（Opportunity Finder）

#### 3.1 产品定位

绝大多数保研学生的择校思路是「只盯头部」——人人都申请清北复交的热门方向，导致激烈内卷。信息差发现器帮助用户找到那些**竞争相对较低但匹配度高的"蓝海"院校/项目**，是一个真正帮用户「弯道超车」的功能。

#### 3.2 核心逻辑

```
输入:
  ├── 用户画像（五维分数 + 本科院校 + 专业方向）
  ├── 用户已关注的院校列表
  └── 全量院校/项目数据库

处理:
  ├── 1. 根据用户画像筛选「匹配度 > 阈值」的院校/项目
  ├── 2. 计算每个项目的「竞争热度」指标
  │   ├── 平台内收藏/关注人数
  │   ├── 往年报录比（如有数据）
  │   ├── 院校知名度排名
  │   └── 该项目在社交媒体的讨论热度（可选）
  ├── 3. 计算「信息差指数」= 匹配度 / 竞争热度
  └── 4. 按信息差指数降序排列，推荐 Top N

输出:
  ├── 推荐院校/项目卡片列表
  ├── 每张卡片包含：匹配度、竞争热度、信息差指数
  ├── AI 推荐理由（为什么这个项目适合你但很少人关注）
  └── 快捷操作：收藏、查看详情、加入时间线
```

#### 3.3 推荐理由示例

> **华东师范大学 · 计算机科学与技术**
>
> 匹配度 82% | 竞争热度 中低 | 信息差指数 ★★★★☆
>

### 功能四：文书 AI 润色（Document AI Polish）

#### 4.1 产品定位

保研申请中除了简历还需要提交：个人陈述（Personal Statement）、研究计划（Research Proposal）、推荐信草稿等文书。这些文书的质量直接影响录取结果，但大多数本科生缺乏学术写作经验。文书 AI 润色提供一键式的智能改写服务。

#### 4.2 支持的文书类型

| 文书类型 | 典型长度 | 润色重点 |
|----------|----------|----------|
| 个人陈述 | 800-1500 字 | 结构优化、学术化表达、动机阐述、逻辑连贯性 |
| 研究计划 | 500-1000 字 | 研究问题明确性、方法论表述、创新点突出、文献引用规范 |
| 推荐信草稿 | 300-600 字 | 语气和视角调整（第三人称）、具体事例补充、推荐力度强化 |
| 自我评价 | 200-400 字 | 精练、避免空话、突出差异化优势 |
| 邮件（联系导师） | 100-300 字 | 礼貌得体、表达清晰、展示对导师研究方向的了解 |

#### 4.3 润色流程

```
步骤 1: 选择文书类型
  用户选择要润色的文书类型（个人陈述 / 研究计划 / 推荐信 / 邮件 / 自定义）

步骤 2: 输入原文
  ├── 方式 A: 直接在文本编辑器中输入/粘贴
  ├── 方式 B: 上传 Word/PDF 文件，自动提取文本
  └── 方式 C: 从简历工坊导入已有内容（如自我评价）

步骤 3: 补充上下文（可选但推荐）
  ├── 目标院校和专业
  ├── 目标导师（如有）
  └── 特别想强调的方面

步骤 4: AI 润色
  ├── 调用 LLM，根据文书类型使用对应的 system prompt
  ├── 输出润色版本 + 逐段修改说明
  └── 支持「润色程度」选择：轻度润色 / 深度改写 / 学术化增强

步骤 5: 对比与编辑
  ├── 左右对比视图：原文 vs 润色版
  ├── 差异高亮：新增（绿）、删除（红）、修改（蓝）
  ├── 逐条修改建议，可单独接受/拒绝
  └── 支持手动编辑润色版

步骤 6: 导出
  ├── 复制到剪贴板
  ├── 下载为 Word/PDF
  └── 保存到「我的文书」列表（可多版本管理）
```

#### 4.4 Prompt 工程示例（个人陈述润色）

```
你是一位资深的研究生申请文书顾问，擅长帮助中国本科生撰写保研申请的个人陈述。

请根据以下信息润色这份个人陈述：
- 目标院校：{target_school}
- 目标专业：{target_major}
- 润色程度：{polish_level}

润色要求：
1. 保持原文核心内容不变，不编造事实
2. 优化文章结构（开头引人入胜 → 学术背景 → 科研经历 → 未来规划 → 结尾）
3. 将口语化表达改为学术化、正式的书面语
4. 增强逻辑连贯性，使用恰当的过渡词
5. 突出与目标专业的匹配度
6. 控制在 {word_limit} 字以内

输出格式：
1. 先输出完整的润色版本
2. 然后逐段列出修改说明，格式为：
   [段落N] 原文摘要 → 修改内容 | 修改理由
```

#### 4.5 技术方案

| 层级 | 方案 |
|------|------|
| 前端路由 | `/ai/document` 或 `/ai/polish`，独立于简历工坊 |
| 编辑器 | 使用 `textarea` + diff 高亮（轻量方案）；后续可引入 `tiptap` 或 `lexical` 富文本编辑器 |
| Diff 展示 | `diff-match-patch` 库或手动实现段落级 diff |
| LLM 调用 | 后端 `/api/ai/polish` 接口，流式返回润色结果（SSE） |
| 版本管理 | 用户表关联 `user_documents` 表：`{ id, user_id, type, title, content, polished_content, version, created_at }` |
| 文件解析 | 后端使用 `python-docx` / `PyMuPDF` 提取上传文件的文本 |

#### 4.6 MVP 范围

第一版：
- 支持个人陈述和联系导师邮件两种类型
- 文本输入 + AI 润色 + 左右对比
- 复制导出

后续迭代：
- 更多文书类型
- 文件上传解析
- 版本管理
- 逐条接受/拒绝修改

### 功能五：面试问题预测（Interview Predictor）

#### 5.1 产品定位

当前的 AI 模拟面试功能使用通用题库。面试问题预测则是根据用户**个人简历内容** + **目标院校/导师的研究方向**，生成高度定制化的"你可能会被问到的问题"及参考答案，帮助用户进行针对性准备。

#### 5.2 预测逻辑

```
输入:
  ├── 用户简历数据（从简历工坊读取）
  │   ├── 科研经历 → 生成科研深挖问题
  │   ├── 论文成果 → 生成论文细节问题
  │   ├── 竞赛获奖 → 生成竞赛相关问题
  │   ├── 技能标签 → 生成技术问题
  │   └── 自我评价 → 生成动机类问题
  ├── 目标导师信息（从导师库读取）
  │   ├── 研究方向 → 生成方向匹配问题
  │   └── 近期论文 → 生成论文阅读问题
  └── 目标院校/专业
      ├── 面试风格（结构化/半结构化/自由）
      └── 往年面试问题（社区回报数据）

LLM 处理:
  ├── 根据以上信息生成 15-25 个预测问题
  ├── 每个问题标注：
  │   ├── 问题类别（专业知识/科研经历/动机/英语/综合）
  │   ├── 预测概率（高/中/低）
  │   ├── 难度等级（★~★★★★★）
  │   └── 参考答案 + 回答要点
  └── 按「预测概率 × 重要性」排序

输出:
  ├── 问题列表（分类展示）
  ├── 每题可展开查看参考答案
  ├── 可一键进入模拟面试练习该问题
  └── 标记「已准备」/「需要练习」状态
```

#### 5.3 问题类型矩阵

| 问题类别 | 生成依据 | 示例 |
|----------|----------|------|
| **科研深挖** | 简历中的每段科研经历 | "你在 XX 项目中具体负责哪部分工作？遇到最大的技术挑战是什么？" |
| **论文细节** | 简历中的每篇论文 | "请简要介绍你这篇论文的创新点。审稿人提了哪些主要问题？" |
| **方向匹配** | 目标导师研究方向 | "你对 XX 方向有什么了解？读过这个方向哪些经典论文？" |
| **专业基础** | 用户专业 + 目标专业 | "请解释一下 XX 算法的基本原理和时间复杂度。" |
| **动机类** | 目标院校 + 用户背景 | "为什么选择我们学校/课题组？你的研究计划是什么？" |
| **英语问答** | 固定 + 个性化 | "Please introduce your research experience in English." |
| **压力/综合** | 简历中的弱项 | "你的论文数量偏少，你觉得自己在科研方面有竞争力吗？" |

#### 5.4 UI 设计

- 路由：`/ai/interview/predict` 或嵌入 `/ai/interview` 作为「问题预测」Tab
- 顶部选择目标院校/导师，点击「生成预测」
- 问题列表按类别分组，可折叠/展开
- 每个问题卡片：问题文本、类别标签、概率标签、难度星级、展开参考答案、操作按钮（标记状态 / 进入模拟练习）
- 底部统计：已准备 X/N 题、建议重点准备的 Top 5

#### 5.5 技术方案

| 层级 | 方案 |
|------|------|
| 数据获取 | 从简历工坊 store 读取用户简历数据；从导师库 API 获取目标导师信息 |
| LLM 调用 | 后端 `/api/ai/interview/predict` 接口，输入简历 JSON + 导师信息 + 院校 |
| Prompt | 结构化 prompt，要求 LLM 输出 JSON 格式的问题列表 |
| 缓存 | 相同输入条件下缓存结果 24 小时，避免重复调用 |
| 与模拟面试联动 | 预测问题可一键导入为模拟面试的自定义题库 |
| 社区反馈 | 用户面试后回报「确实被问到」，用于后续优化预测模型 |

#### 5.6 MVP 范围

第一版：
- 根据简历数据 + 目标院校生成预测问题（不含导师匹配）
- 问题列表 + 参考答案展示
- 标记准备状态

后续迭代：
- 导师研究方向匹配
- 与模拟面试联动
- 社区反馈改进预测准确度
- 往年真题融合
