## 一、如何运行和查看界面

### 1.1 环境要求

| 工具    | 版本要求   | 说明                     |
| ------- | ---------- | ------------------------ |
| Node.js | ≥ 18.x    | 推荐 20.x LTS           |
| pnpm    | ≥ 9.x     | 包管理器（已全局安装）    |

### 1.2 启动步骤

```bash
# 1. 进入前端项目目录
cd /Users/cedric/project/yantu/frontend

# 2. 安装依赖（首次或依赖变更后执行）
pnpm install

# 3. 启动开发服务器
pnpm dev
```

启动后访问 **http://localhost:3000** 即可查看界面。

## P9 — 脑图二级连线修复 + 多云/雾天效果彻底重做

### 修改时间：2026-03-31

### 1. 脑图二级连线精确对齐

**文件：** `src/components/graph/ProjectMindMap.tsx`

- **问题**：二级连线只画了一条线到子节点容器的中心点，而不是分别连接到每个叶子节点
- **根因**：之前为每个分支的子节点容器整体创建一个 ref，只画一条线到容器中心
- **解决方案**：
  - 为**每个叶子节点**创建独立的 ref（使用 `Map<string, HTMLDivElement>` 存储，key 格式如 `L-0-2` 表示左侧第0个分支的第2个叶子）
  - 为**每个三级子节点**也创建独立的 ref（key 格式如 `R-0-3-1`）
  - 从分支标签分别画贝塞尔曲线到**每个**叶子节点
  - 从有子节点的叶子再画线到每个三级子节点
  - 连线层级：根→分支标签（2.5px）→ 叶子（1.8px）→ 三级节点（1.2px）
  - 使用 `useEffect` + 双延迟测量（50ms + 200ms）确保 DOM 完全渲染后再绘制

### 2. 多云效果：圆形 div → box-shadow 云朵

**文件：** `src/components/layout/DynamicBackground.tsx`

- **问题**：之前的圆形 div 尺寸用 px 单位（120px × 80px），在全屏背景下几乎不可见，看起来只有灰色
- **解决方案**：
  - 每朵云用**一个圆形 div + 多个 box-shadow** 构建蓬松形态
  - 所有尺寸使用 `vw` 单位（如 `8vw`、`10vw`），确保在任何屏幕上都清晰可见
  - box-shadow 不增加 DOM 元素，性能极好（5 朵云 = 5 个 DOM）
  - 每个 shadow 偏移不同位置和大小，组合出自然的云朵轮廓
  - 5 朵云不同速度（32-52s）和延迟，从左到右循环飘动

### 3. 雾天效果：linear-gradient 方块 → SVG 纹理平铺飘动

**文件：** `src/components/layout/DynamicBackground.tsx`

- **问题**：之前的雾天用 `linear-gradient` 在固定宽度 div 上做 `alternate` 来回位移，看起来是静止的渐变方块
- **解决方案**：
  - 用 SVG 生成柔和的椭圆雾气纹理（多个随机大小/位置的 `<ellipse>`）
  - 作为 `background-image` 平铺在 300% 宽度的容器上
  - 使用 `translateX(33.33%)` 的 `linear infinite` 动画实现**无缝循环飘动**
  - 3 层雾气纹理，不同速度（14s/18s/25s）和方向（正/反向）
  - 底部浓雾 + 呼吸感覆盖层增加层次
