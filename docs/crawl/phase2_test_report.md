# 阶段二测试报告 - 信息发布页定位

> 测试日期：2026-03-27
> 测试范围：阶段二通知页定位模块 - 六层策略级联 + 验证评分 + 批量处理
> 测试结果：**优化后覆盖率 97.1%（37校72院→69有效院），核心功能验证通过**

## 一、功能概述

### 1.1 六层策略级联

阶段二的核心是 `notice_page_locator.py`，实现了六层策略级联定位学院通知列表页：

| 策略 | 方法 | 说明 | 适用场景 |
|------|------|------|---------|
| 策略1 | 导航关键词匹配 | 在学院首页导航栏中查找"招生"/"通知"/"新闻"等链接 | 大部分高校（~70%） |
| 策略2 | 全文链接扫描 + URL 路径特征 | 扫描页面所有链接，匹配 `/tzgg`、`/zsxx` 等路径模式 | 导航结构不规范的高校 |
| 策略3 | Playwright 渲染 | 浏览器渲染后扫描完整 DOM | JS 动态加载的导航 |
| 策略4 | LLM 分析页面结构 | 将精简 HTML 发送给 LLM 智能定位 | 结构特殊的页面 |
| 策略5 | 子路径猜测 | 尝试常见通知页路径（如 `/tzgg/index.htm`） | 所有策略失败时 |
| 策略6 | 研究生院回退 | 使用学校研究生院的通知页 | 学院无独立通知页 |

### 1.2 验证评分机制

每个候选通知页经过自动验证，满分 100 分：

| 评分项 | 分值 | 说明 |
|--------|------|------|
| 带日期的链接条目 ≥ 5 条 | +40 | 通知列表页的核心特征 |
| 标题长度合理（8-150字符） | +20 | 排除导航链接等噪声 |
| 日期格式符合常见模式 | +20 | YYYY-MM-DD 等格式 |
| 页面标题含关键词 | +10 | "通知"/"公告"/"招生"等 |
| 存在分页组件 | +10 | "下一页"/">>"/分页 class |

≥ 30 分通过验证。

### 1.3 多信息源策略

每个学院最多记录 3 个信息源，按类型优先级排序：
1. **招生页**（最高优先级）：研究生招生、推免等
2. **通知页**：通知公告、学院公告等
3. **新闻页**：新闻动态、学院新闻等

### 5.3 后续优化方向

1. **并发优化**：不同域名的请求可以并发执行
2. **LLM 辅助**：对验证分数 30-60 的候选页面，使用 LLM 二次确认
3. **反爬绕过**：对有反爬保护的网站，使用代理或浏览器指纹模拟
4. **网站不稳定重试**：对临时不可用的网站，增加延迟重试机制
